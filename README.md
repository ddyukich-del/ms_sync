# МойСклад Sync для ocStore/OpenCart 4.1

Версия: **1.0.1**

Автор: **d_dyuk**

Модуль синхронизирует интернет-магазин на ocStore/OpenCart 4.1 с МойСклад.

## Статус релиза

Версия **1.0.1** — стабильный релиз с исправлением безопасного обновления остатков для товаров из заказов поставщикам. Перед выпуском зафиксирована стабильная база 0.99.0 RC: синхронизация товаров, категорий, остатков, изображений, заказов поставщикам, dashboard-виджет, диагностика и безопасная очистка старых экспериментальных OCMOD/event-записей.


## Основные возможности

- импорт групп товаров МойСклад как категорий ocStore;
- импорт товаров только по выбранным складам;
- поддержка нескольких складов одновременно;
- синхронизация фактических остатков;
- учет заказов поставщикам по нескольким выбранным статусам;
- раздельное отображение фактического остатка и ожидаемого количества от поставщика;
- опциональное объединение товаров из заказов поставщикам с товарами в наличии по нормализованному наименованию;
- отдельная загрузка изображений товаров;
- вкладка «Товары» с фильтрами, сортировкой и CSV-экспортом;
- пакетная обработка задач для слабого хостинга;
- логи операций и диагностический API-лог;
- вкладка «Диагностика» для проверки установки, прав, dashboard-виджета и доступности папок.


## Диагностика и релизная проверка

С версии **0.99.0** добавлена вкладка «Диагностика». Она помогает быстро проверить:

- включен ли модуль;
- указан ли API-токен;
- выбраны ли склады и тип цены;
- зарегистрирован ли dashboard-виджет;
- есть ли права у текущей группы администратора;
- доступна ли папка `image/` для записи;
- доступна ли папка `storage/logs/`;
- создан ли API debug log и какой у него размер;
- какая задача синхронизации запускалась последней и были ли ошибки.

API debug log теперь управляется настройкой. Для обычной работы его лучше держать выключенным и включать только на время отладки. Токен МойСклад в лог не пишется.

## Dashboard-виджет

С версии **0.98.0** модуль добавляет штатный dashboard-виджет «МойСклад» на главную страницу админки.

Виджет сделан через стандартный тип расширения OpenCart/ocStore `dashboard`:

- без OCMOD;
- без правки левого меню;
- без изменения core-файлов CMS.

Виджет показывает:

- всего товаров в интеграции;
- товаров в наличии;
- товаров из заказов поставщикам;
- ожидаемое количество от поставщика;
- товары вне актуальной выгрузки;
- дату последней синхронизации.

Также в виджете есть быстрые кнопки:

- «Открыть список товаров»;
- «Синхронизация»;
- «Диагностика»;
- «Настройки».

Если после установки виджет не появился на главной странице, откройте страницу настроек модуля один раз. При открытии модуль мягко регистрирует dashboard-расширение и права доступа для текущей группы администратора. После этого обновите главную страницу админки.



### Исправления 0.98.1

В 0.98.1 усилена регистрация dashboard-виджета при обновлении поверх уже установленного модуля:

- настройки `dashboard_moysklad_sync_*` проверяются напрямую в таблице `setting`;
- права на route `extension/moysklad_sync/dashboard/moysklad_sync` добавляются без дублей;
- добавлены стандартные методы `install()` / `uninstall()` / `index()` у dashboard-контроллера;
- добавлен языковой ключ для списка dashboard-расширений.

Если после установки настройки и запись в `extension` уже есть, но виджет не выводится, откройте страницу модуля один раз и затем обновите главную страницу админки. Если не поможет — проверьте права группы администратора на route `extension/moysklad_sync/dashboard/moysklad_sync`.

## Почему нет пункта в левом меню

В версии 0.98.x модуль сознательно **не добавляет пункт в левое меню админки**.

Причина: в OpenCart/ocStore 4 левое меню формируется внутри `common/column_left.php`, и универсального безопасного API для добавления пункта из обычного module-extension нет. OCMOD-правки меню оказались хрупкими на разных сборках ocStore и могли мешать установке/удалению.

Быстрый доступ реализован через dashboard-виджет. Это безопаснее для коммерческой эксплуатации.

## Установка

1. Загрузите архив `moysklad_sync.ocmod.zip` через установщик расширений.
2. Нажмите Install у загруженного архива.
3. Откройте модуль в разделе расширений.
4. Укажите API-токен МойСклад.
5. Нажмите «Загрузить склады, цены и статусы закупок».
6. Выберите склады, тип цены и статусы заказов поставщикам.
7. Сохраните настройки.
8. Запустите импорт.

## Рекомендации для первого запуска

- сначала используйте маленькие батчи;
- опасные действия вроде удаления товаров лучше включать только после тестового прогона;
- для ожидаемых товаров безопаснее оставлять `quantity = 0`, а ожидаемое количество смотреть во вкладке «Товары»;
- перед установкой на боевой сайт сделайте резервную копию файлов и базы.

## Разделение фактического и ожидаемого количества

При объединении товара из заказа поставщику с товаром в наличии модуль не прибавляет ожидаемое количество к стандартному полю `product.quantity`.

- `product.quantity` = только фактический остаток по выбранным складам;
- `incoming_quantity` = ожидаемое количество из заказов поставщикам;
- во вкладке «Товары» эти значения показываются отдельными колонками.

Это нужно, чтобы менеджер понимал, что реально можно отгрузить сейчас, а что еще едет на склад.
