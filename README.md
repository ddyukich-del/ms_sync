# МойСклад Sync для ocStore/OpenCart 4

Версия: `0.9.10`



## Важное изменение 0.9.9

Если товары уже были импортированы ранее, но связи с категориями отсутствовали, повторный импорт теперь восстанавливает записи в `product_to_category` даже для товаров, которые не изменились и были бы пропущены по хэшу.

## Важное изменение 0.9.6

Добавлен резервный способ создания категорий по `pathName`. Это нужно для аккаунтов МойСклад, где товары успешно приходят из отчета остатков выбранного склада, но карточка товара не возвращает полноценный `productFolder`. В таком случае модуль строит цепочку категорий по строковому пути, например `Родитель/Дочерняя`, и привязывает товар к последней категории.

## Важное изменение 0.9.3

Категории больше не импортируются всем справочником МойСклад перед товарами. При полном импорте модуль берет товары из остатков выбранного склада и создает только категории этих товаров и их родителей. Это исправляет ситуацию, когда категории были удалены вручную, связи в `moysklad_category_link` остались, а новые категории не пересоздавались.

Ошибки по отдельным товарам и категориям теперь отображаются во вкладке логов вместе с обычными событиями.

## Важное изменение 0.9.1

Если категории были удалены вручную из админки, но связи с МойСклад остались в служебной таблице `moysklad_category_link`, модуль теперь не считает такие категории существующими. Он создает их заново и обновляет связь на новый `category_id`.

## Важное изменение 0.9.0

Полный импорт товаров теперь работает строго от выбранного склада.

Раньше модуль сначала читал весь каталог `/entity/product`, поэтому на сайт могли попасть товары, которые существуют в МойСклад, но не лежат на выбранном складе. В версии 0.9.0 это исправлено: список импортируемых товаров берется из отчета остатков выбранного склада `/report/stock/bystore` с фильтром по складу.

Алгоритм полного импорта:

1. Импортировать группы/категории МойСклад.
2. Построить дерево категорий.
3. Получить отчет остатков выбранного склада.
4. Для каждой строки отчета с положительным остатком получить полную карточку товара.
5. Создать или обновить товар на сайте.
6. Записать quantity равным остатку выбранного склада.
7. Товары, которые связаны с МойСклад, но не встретились в текущем складском отчете, обработать по настройке отсутствующих товаров.

## Рекомендация после обновления с ранних тестовых версий

Если ранние версии уже успели создать товары с других складов, для очистки тестовой базы можно временно поставить настройку:

- товары, отсутствующие в МойСклад/выбранном складе: `Удалять`

Затем запустить импорт заново. После очистки безопаснее вернуть режим `Отключать`.

Если товары нельзя удалять, оставьте `Отключать` — лишние товары станут неактивными, но останутся в админке.

## Установка

1. Загрузить архив `moysklad_sync.ocmod.zip` через установщик расширений.
2. Нажать Install у загруженного архива.
3. Установить модуль в разделе модулей/расширений.
4. Проверить права группы администратора на маршрут `extension/moysklad_sync/module/moysklad_sync`.
5. Открыть настройки модуля.
6. Ввести API-токен МойСклад.
7. Загрузить склады и типы цен.
8. Выбрать один склад и тип цены.
9. Сохранить настройки.
10. Запустить импорт товаров.

## Рекомендуемые батчи для слабого сервера

- Категории: 30
- Товары: 10–20
- Остатки: 50
- Изображения: 3

## Что делает модуль

- Импортирует категории.
- Импортирует товары только выбранного склада.
- Синхронизирует цену выбранного типа цены.
- Синхронизирует количество по выбранному складу.
- Отключает или удаляет товары вне выбранного склада согласно настройкам.
- Загружает изображения отдельной кнопкой.
- Ведет задачи и логи.


## Версия 0.9.6

Исправлена ошибка `preg_split(): Unknown modifier ]` при создании категорий по `pathName`. После установки нужно заново запустить импорт товаров, чтобы модуль повторно прошел товары выбранного склада и создал категории.


### Важно про категории с версии 0.9.6

Категории снова синхронизируются из справочника групп товаров МойСклад перед импортом товаров. Это сделано намеренно: товары остаются строго ограничены выбранным складом и ненулевым остатком, но справочник категорий должен восстанавливаться надежно даже если карточка товара не возвращает `productFolder` или `pathName`.


## Диагностика API МойСклад

Начиная с версии 0.9.8 модуль пишет реальные ответы API МойСклад в файл:

```text
storage/logs/moysklad_sync_api_debug.log
```

В файл попадают метод, URL, HTTP-статус, заголовки ответа и тело ответа. API-токен в файл не записывается. Если файл становится больше 20 МБ, он ротируется в `moysklad_sync_api_debug.log.old`.

Категории ocStore импортируются из групп товаров МойСклад — endpoint `entity/productfolder`. Этот импорт не ограничивается выбранным складом. Выбранный склад ограничивает только список товаров: на сайт попадают только товары выбранного склада с остатком больше 0.
