# Changelog

## 0.9.10

- Исправлена ошибка загрузки изображений `Array to string conversion` в `MoyskladClient`.
- Нормализация изображений теперь безопасно разбирает поля `downloadHref`, `href`, `url`, `miniature`, `tiny`, даже если МойСклад возвращает их массивами.
- Warning больше не должен попадать в AJAX-ответ и ломать JSON при нажатии «Загрузить картинки».

## 0.9.9

- Исправлено восстановление связей `product_to_category` для товаров, которые пропускались по неизменному хэшу.
- Теперь при повторном импорте товар, уже существующий на сайте, заново привязывается к категории, если категория была пересоздана после ручного удаления или прошлых тестовых запусков.
- Связь восстанавливается только если у товара есть ID группы МойСклад, чтобы не удалять существующие связи без причины.

## 0.9.8

- Добавлена запись всех API-ответов МойСклад в `storage/logs/moysklad_sync_api_debug.log`.
- Категории импортируются напрямую из `entity/productfolder` без ограничения выбранным складом.
- На время диагностики отключено автоматическое удаление/отключение категорий в конце полного импорта, чтобы модуль не удалял категории, пока мы проверяем реальный ответ API.
- В лог задачи добавлена строка с количеством полученных `productfolder` на каждой странице.

# Changelog

## 0.9.6

- Исправлена PHP-warning ошибка `preg_split(): Unknown modifier ]` при создании категорий по `pathName`.
- Создание категорий по пути теперь не использует регулярные выражения: путь нормализуется через `str_replace` и разбивается через `explode`.
- Добавлена поддержка разделителей `/`, `\` и `>` в `pathName`.
- Эта правка предотвращает попадание warning HTML в AJAX-ответ вместо JSON.

## 0.9.4

- Добавлен fallback создания категорий по `pathName`, если API МойСклад не возвращает `productFolder` в карточке товара или отчете остатков.
- Для fallback-категорий используются стабильные виртуальные ID `path:<sha1>`, поэтому повторный импорт не создает дубли.
- Улучшено извлечение ID категории из раскрытого `productFolder`, если объект пришел с `id`, но без `meta.href`.
- Товар теперь получает категорию, созданную по `pathName`, а не остается без привязки.
- Добавлены комментарии к логике fallback-категорий.

## 0.9.3

- Исправлена логика полного импорта: товары, не встреченные в положительном остатке выбранного склада, теперь обрабатываются по настройке нулевого остатка.
- Импорт карточек товара теперь запрашивает `expand=productFolder`, чтобы надежнее создавать категории товара и родителей.
- Добавлен fallback создания категории из раскрытого `productFolder` без лишнего API-запроса.
- Если категория отсутствует в карточке товара, модуль пишет предупреждение в лог.
- Батч полного импорта теперь использует настройку батча товаров, а не категорий.

## 0.9.2

- Изменен алгоритм категорий: при импорте по выбранному складу создаются только категории импортируемых товаров и их родители.
- Добавлен точечный запрос категории МойСклад по ID и восстановление цепочки родителей.
- Полный импорт больше не стартует с импорта всего справочника категорий.
- После обработки товаров модуль обрабатывает категории, не встреченные в выбранном складе.
- Во вкладке логов теперь отображаются записи из таблицы ошибок, чтобы видеть причины счетчика «Ошибок».

## 0.9.1

- Исправлено повторное создание категорий после ручного удаления из админки ocStore.
- Если связь `moysklad_category_link` есть, но строки в `oc_category` уже нет, модуль создает категорию заново и перепривязывает связь к новому `category_id`.
- Это закрывает сценарий, когда категории были удалены вручную, а модуль считал их уже импортированными и не создавал повторно.

## 0.9.0

- Полный импорт товаров больше не идет по всему каталогу `/entity/product`.
- Импорт товаров теперь строится от отчета остатков выбранного склада `/report/stock/bystore`.
- Товары с других складов не создаются при полном импорте.
- Товары с нулевым/отрицательным остатком на выбранном складе не создаются; для уже связанных товаров применяется политика отсутствующего/нулевого остатка.
- Исправлено извлечение ID из `href` МойСклад с query string вида `?expand=supplier`.
- Остатки теперь берутся из `stockByStore`, а не из суммарного/неподходящего поля отчета.
- Категории, в которые импортируются товары выбранного склада, включаются вместе с родительскими категориями.
- Категория теперь принудительно приводит статус к актуальному даже если хэш данных не изменился.

## 0.8.9

- Исправлена ситуация с осиротевшими связями `moysklad_product_link`, когда связь есть, а товара в `oc_product` уже нет.

## 0.8.8

- Исправлен отсутствующий метод `toBool()` в `MoyskladClient`.

## 0.8.7

- Исправлена нормализация boolean-полей МойСклад.
- Убрана зависимость пакетной обработки от `link_id`.

## 0.8.6 и ранее

- Базовый импорт категорий, товаров, остатков, изображений, задачи, логи и админка.


## 0.9.6

- Возвращена полноценная синхронизация справочника категорий перед импортом товаров.
- Товары по-прежнему импортируются только из отчета положительных остатков выбранного склада.
- Исправлен порядок шагов: категории → дерево категорий → товары выбранного склада → удаление/отключение товаров вне выбранного склада → обработка категорий, которых нет в МойСклад.
- Причина правки: в некоторых аккаунтах МойСклад карточка товара/отчет остатков не возвращали `productFolder`/`pathName`, поэтому создание категорий по товарам не срабатывало. Импорт справочника категорий через `/entity/productfolder` восстанавливает категории надежнее.
