# Changelog

## 1.0.2

- Изменена логика заказов поставщикам: позиции сопоставляются с товарами только по стабильному ID номенклатуры МойСклад из `assortment.meta.href`, а не по названию.
- Добавлена настройка «Добавлять ожидаемое в quantity сайта». При включении `product.quantity = фактический остаток + ожидаемое количество из выбранных заказов поставщикам`.
- Фактический остаток и ожидаемое количество продолжают храниться и отображаться отдельно: во вкладке «Товары» добавлена колонка «Количество на сайте».
- Если товара нет на выбранных складах, но он есть в выбранном заказе поставщику, модуль создает/обновляет карточку по полной номенклатурной карточке МойСклад и ставит количество из ожидаемой поставки.
- Старые режимы `separate` и `merge_by_name` оставлены в коде как legacy, но в интерфейсе переведены в неактивный режим. Активный режим — сопоставление по ID МойСклад.
- Кнопка «Обновить остатки» теперь сохраняет ожидаемое количество и при включенной настройке пересчитывает количество сайта как факт + ожидание.
- CSV-экспорт дополнен колонкой «Количество на сайте».

- Версия модуля, `install.json` и User-Agent HTTP-клиента обновлены до 1.0.0.
- Dashboard-виджет, диагностика, CSV-экспорт, работа с несколькими складами, заказы поставщикам и раздельный учет фактического/ожидаемого количества включены в релизную поставку.
- Левое меню админки сознательно не изменяется: быстрый доступ реализован через штатный dashboard-виджет.
